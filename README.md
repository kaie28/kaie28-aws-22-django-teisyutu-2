### 2【概要】

・webアプリ(主にPython、Django)、AWS(Terraform、Docker)、セキュリティ(IAM , Security Group)の3本柱を中心に学習しました。また本番環境と開発環境の各利点とする機能を生かした形でデプロイしました。


### 3【開発背景】

・私の趣味は旅行で、歴史的な人物や建造物などの「裏側にある背景」を学んでロマンを感じることが好きです。この探求する楽しさは、日々愛用している『Instagram』の仕組みへの興味にも共通しています。将来はシステム開発のスキルを活かし、場所を問わず誰もが深く繋がれるコアなコミュニティを数多く創り出したいと思い、今回の構築に挑戦しました。


・私が好きな「Instagram」のアプリをヒントに、プログラミング言語やAWSの選定を行い、全ての工程を独学で行いました。プログラミングは未経験ですが、5か月で完了できるように、事前に勉強内容の順序や学習範囲など入念に計画し、特に「セキュリティ強化」「データ復元の簡易化」「コスト面」を重視して、工夫しながら構築・設計しました。



### 4【工夫した点】

・本番環境ではDockerでの公開と同時進行で、リアルタイムで開発モードからログによるエラー追跡が出来るように設計にしました。

・セキュリティ強化のため、IPやシークレットキーの変数化や、.env　.gitgnoreを利用し対策しました。

・始めの頃は「手動型」で構築していましたが、デプロイ直前で全てデータが消えてしまった苦い経験から、再チャレンジし「全自動型」へ移行して復元しやすい状態に強化しました。
