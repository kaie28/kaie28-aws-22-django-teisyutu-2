### 3【開発背景】

・趣味の旅行や『Instagram』の「裏の背景・仕組み」にロマンを感じ、構築を学びました。手作業での構築の多さを知る中で、ITの現場にも看護師時代に痛感した人手不足と同様の課題があると実感しました。将来は『AWSの自動化』で負担を減らし、時間の余白を作り、そこで新たな開発に挑戦するのが目標です。

・『Instagram』で実際使われている言語や仕組みを参考に、全ての工程を独学で行いました。プログラミングは未経験ですが、5か月で完了できるように、事前に勉強内容の順序や学習範囲など入念に計画し、特に「セキュリティ強化」「データ復元の簡易化」「コスト面」を重視して、工夫しながら構築・設計しました。



### 4【工夫した点】

・本番環境ではDockerでの公開と同時進行で、リアルタイムで開発モードからログによるエラー追跡が出来るように設計にしました。

・セキュリティ強化のため、IPやシークレットキーの変数化や、.env　.gitgnoreを利用し対策しました。

・始めの頃は「手動型」で構築していましたが、デプロイ直前で全てデータが消えてしまった苦い経験から、再チャレンジし「自動型」へ移行して復元しやすい状態に強化しました。
